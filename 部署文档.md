# 物流教学软件 - Docker 部署文档

## 环境要求

- Docker 20.10+
- Docker Compose 1.29+

## 快速开始

### 1. 一键启动（推荐）

在项目根目录执行：

```bash
docker-compose up -d
```

访问：
- 前端：http://localhost:8080
- 后端 API：http://localhost:5000
- Nginx 统一入口：http://localhost (需要单独启动 nginx 服务)

### 2. 单独启动各服务

#### 启动后端

```bash
cd backend
docker build -t logistics-backend .
docker run -d -p 5000:5000 --name logistics-backend logistics-backend
```

#### 启动前端

```bash
cd frontend
docker build -t logistics-frontend .
docker run -d -p 8080:80 --name logistics-frontend logistics-frontend
```

## 服务说明

| 服务名 | 端口 | 说明 |
|--------|------|------|
| backend | 5000 | Flask 后端 API |
| frontend | 8080 | Nginx 托管的前端静态文件 |
| nginx | 80 | (可选) 统一反向代理入口 |

## 常用命令

```bash
# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f

# 停止所有服务
docker-compose down

# 重新构建并启动
docker-compose up -d --build

# 进入后端容器调试
docker-compose exec backend bash

# 进入前端容器调试
docker-compose exec frontend sh
```

## 数据持久化

- SQLite 数据库文件位于 `backend/instance/app.db`
- 通过 volumes 挂载，容器重建后数据不会丢失

## 生产环境建议

1. **修改默认密码**：首次部署后请立即修改 admin 账号密码
2. **使用环境变量**：敏感信息（如密钥）应通过环境变量传入，不要硬编码
3. **配置 HTTPS**：生产环境请配置 SSL 证书
4. **数据库备份**：定期备份 `backend/instance/app.db` 文件

## 故障排查

### 端口冲突

如果端口 5000、8080 或 80 已被占用，请修改 `docker-compose.yml` 中的端口映射。

### 数据库连接失败

检查 `backend/instance` 目录权限，确保容器有读写权限。

### 前端无法访问后端 API

检查 Nginx 配置文件中的 `proxy_pass` 地址是否正确。

## 更新部署

当代码有更新时：

```bash
# 重新构建镜像
docker-compose build --no-cache

# 重启服务
docker-compose up -d
```

## 清理

```bash
# 停止并删除容器、网络
docker-compose down

# 删除镜像
docker rmi logistics-backend logistics-frontend

# 删除 volumes（会删除数据库！）
docker-compose down -v
```
