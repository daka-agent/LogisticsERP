# 物流教学软件 - 方案设计文档

> **版本：** v1.0
> **日期：** 2026-05-01
> **目标：** 为本科物流管理专业学生设计一套可在浏览器运行的小型物流企业信息系统仿真教学软件
> **覆盖范围：** 采购管理 + 运输管理 + 仓储管理 + 库存管理，支持单人练习 + 多人协作角色扮演

---

## 一、项目背景与目标

### 1.1 项目背景

物流管理专业的教学中，学生需要理解物流企业的实际业务运作流程，但传统课堂教学难以提供真实的操作体验。本软件旨在模拟小型物流企业的核心信息系统，让学生在课堂上进行模拟操作，体验从采购、运输、仓储到库存管理的完整业务流程。

### 1.2 项目目标

| 目标维度 | 具体要求 |
|---------|---------|
| 教学目标 | 让学生理解并实操物流企业核心业务流程 |
| 功能目标 | 覆盖采购、运输、仓储、库存四大业务模块 |
| 体验目标 | 支持单人练习和多人角色扮演两种教学模式 |
| 技术目标 | 网页版（H5），浏览器直接打开，适合机房教学 |
| 部署目标 | 教师可一键部署，学生无需安装任何软件 |

### 1.3 目标用户

| 用户角色 | 使用场景 |
|---------|---------|
| 本科物流管理专业学生 | 课堂实操练习、课后复习 |
| 物流专业教师 | 教学演示、进度监控、评分考核 |
| 教学管理员 | 系统维护、数据备份 |

## 二、技术选型

### 2.1 技术栈总览

| 层次 | 技术方案 | 选择理由 |
|------|---------|---------|
| **前端框架** | Vue 3 + Vite | 组合式 API、组件化开发、Vite 构建速度快 |
| **UI 组件库** | Element Plus | 中文文档完善、中后台组件齐全、社区活跃 |
| **数据可视化** | ECharts | 图表类型丰富、性能优异 |
| **后端框架** | Python Flask | 轻量、易学、适合教学演示、Python 生态成熟 |
| **ORM** | SQLAlchemy | 功能强大、支持多种数据库、Python 标配 |
| **实时通信** | Flask-SocketIO | 与 Flask 无缝集成，支持 WebSocket |
| **用户认证** | Flask-Login | Session 管理、简单可靠 |
| **数据库** | SQLite（开发）/ MySQL（生产） | 开发零配置，生产可切换 |
| **部署方案** | Docker / 纯 Python 启动 | 灵活，适应不同机房环境 |

### 3.2 单人练习模式架构

学生以单人身份登录，可自由切换操作不同模块，体验完整的业务流程：
- 学生扮演所有角色（采购专员、客服、调度员、仓库管理员、司机）
- 系统提供流程引导，提示下一步操作
- 操作日志全程记录，方便回看和评分

## 四、功能模块详细设计

### 4.1 采购管理模块

#### 4.1.1 业务流程

```
采购专员填写采购申请
       ↓
  系统生成申请单（待审批）
       ↓
  教师/管理员审批
       ↓
  审批通过 → 选择供应商 → 生成采购订单
       ↓
  供应商发货 → 到货验收（核对数量/质量）
       ↓
  验收通过 → 自动生成入库单
       ↓
  仓库管理员上架 → 库存增加
```

#### 4.1.2 功能清单

| 功能 | 描述 | 操作说明 |
|------|------|---------|
| 采购申请 | 采购专员发起采购需求 | 填写商品名称、数量、期望单价、申请原因 |
| 采购审批 | 审批人审核采购申请 | 查看详情 → 同意/驳回（附理由）|
| 供应商选择 | 为已审批的申请选择供应商 | 查看供应商列表、评分、历史报价 |
| 采购订单 | 生成正式采购订单 | 确认供应商、商品明细、总金额 |
| 到货验收 | 记录到货情况 | 核对数量、质量检查（合格/不合格）|
| 采购入库 | 验收通过后生成入库单 | 自动流转到仓储管理模块 |
| 采购报表 | 采购数据分析 | 按供应商/时间段/商品分类汇总 |

#### 4.1.3 教学要点
- 理解采购需求产生的原因（库存不足/客户订单驱动）
- 掌握采购审批流程（内控机制）
- 学会比较和选择供应商（价格、质量、交期）
- 理解采购与仓储的衔接关系

---

### 4.2 运输管理模块

#### 4.2.1 业务流程

```
客服录入运输订单（客户信息、货物信息、起终点）
       ↓
  系统生成订单（待审核）
       ↓
  调度员审核订单
       ↓
  审核通过 → 车辆调度（选择车型、指派司机）
       ↓
  司机确认接单 → 开始运输
       ↓
  运输中（可更新位置和状态）
       ↓
  到达目的地 → 客户签收
       ↓
  运费结算
```

#### 4.2.2 功能清单

| 功能 | 描述 | 操作说明 |
|------|------|---------|
| 订单录入 | 客服录入客户运输需求 | 客户信息、货物名称/重量/体积、起终点 |
| 订单审核 | 调度员审核订单信息 | 检查货物合规性、确认收发货信息 |
| 车辆调度 | 分配车辆和司机 | 选择可用车型、指派空闲司机、规划路线 |
| 运输跟踪 | 实时查看运输状态 | 在途/到达/异常，可模拟更新位置 |
| 签收确认 | 记录客户签收 | 上传签收单、确认收货 |
| 运费结算 | 计算并确认运费 | 按重量/体积/距离自动计算，支持手动调整 |
| 运输报表 | 运输数据分析 | 准时率、车辆利用率、运费收入 |

#### 4.2.3 教学要点
- 理解运输订单的全生命周期管理
- 掌握车辆调度的基本原则（车型匹配、满载率）
- 理解运输成本构成和计算方法
- 体验从接单到签收的完整运输流程

### 4.3 仓储管理模块

#### 4.3.1 业务流程

**入库流程：**
```
收到入库通知（采购入库/调拨入库/退货入库）
       ↓
  预约入库（确认时间、仓库）
       ↓
  实物验收（核对数量、质检）
       ↓
  系统上架（推荐货位 or 手动选择）
       ↓
  确认入库 → 库存增加
```

**出库流程：**
```
收到出库通知（销售出库/调拨出库/报损出库）
       ↓
  生成出库单（分配库存/货位）
       ↓
  拣货（按出库单到指定货位拣货）
       ↓
  打包复核（核对品种和数量）
       ↓
  发货确认 → 库存减少
```

#### 4.3.2 功能清单

| 功能 | 描述 | 操作说明 |
|------|------|---------|
| 仓库管理 | 维护仓库/库区/货位 | 三级结构：仓库 → 库区 → 货位 |
| 入库管理 | 处理各类入库业务 | 采购入库、退货入库、调拨入库 |
| 出库管理 | 处理各类出库业务 | 销售出库、调拨出库、报损出库 |
| 库内作业 | 货位调整、盘点 | 移库（调整货位）、盘点（账实对比）|
| 仓库报表 | 仓库利用率分析 | 库位占用率、出入库频率 |

#### 4.3.3 教学要点
- 理解仓库的分区管理和货位编码规则
- 掌握入库验收和出库拣货的标准操作
- 理解仓库作业的基本规范

---

### 4.4 库存管理模块

#### 4.4.1 功能清单

| 功能 | 描述 | 操作说明 |
|------|------|---------|
| 库存查询 | 多维度查询当前库存 | 按商品/仓库/货位/批次查询 |
| 库存预警 | 自动监控库存水平 | 低于最低库存或高于最高库存时预警 |
| 库存流水 | 查看所有库存变动 | 每笔出入库操作记录，可追溯 |
| 盘点管理 | 定期或不定期盘点 | 创建任务 → 实盘录入 → 差异报告 |
| 库存报表 | 库存分析 | 周转率、呆滞库存、库龄分析 |

#### 4.4.2 教学要点
- 理解库存预警机制的作用
- 掌握盘点操作流程（盲盘/明盘）
- 学会阅读和分析库存报表

---

### 4.5 教学辅助功能

#### 4.5.1 操作日志与回放
- 记录每个学生的每一步操作（时间、动作、数据变更）
- 教师可按时间线回放学生的操作过程
- 用于评分参考和错误纠正

#### 4.5.2 流程可视化
- 用步骤条（Stepper）展示当前订单所处的业务环节
- 用流程图（Flowchart）展示完整的业务流程全景
- 高亮当前步骤，灰色显示已完成和未开始步骤

#### 4.5.3 自动评分系统
- 评分维度：操作正确性、流程完整性、时效性、规范性
- 扣分项：操作顺序错误、数据填写不规范、超时未处理
- 加分项：高效处理、主动发现问题

#### 4.5.4 教学场景预设
教师可预设不同的教学场景，增加教学趣味性和挑战性：

| 场景名称 | 场景描述 | 考察要点 |
|---------|---------|---------|
| 基础流程 | 正常的单据流转 | 基本操作能力 |
| 高峰期 | 大量订单涌入 | 分优先级处理能力 |
| 车辆故障 | 某车辆在途中故障 | 应急调度能力 |
| 供应商延迟 | 采购到货延迟 | 库存预警和替代方案 |
| 爆仓 | 仓库库存超出容量 | 库存管理和紧急出库 |
| 质量问题 | 到货验收不合格 | 退货和重新采购流程 |

## 五、角色扮演设计（多人协作模式）

### 5.1 角色定义

| 角色 | 核心职责 | 可操作模块 |
|------|---------|-----------|
| 采购专员 | 管理采购需求，跟踪供应商 | 采购管理 |
| 客服 | 接收客户需求，录入订单 | 运输管理（订单）|
| 调度员 | 分配运输资源，规划路线 | 运输管理（调度）|
| 仓库管理员 | 管理出入库和库存 | 仓储管理 + 库存管理 |
| 司机 | 执行运输任务 | 运输管理（执行）|
| 教师 | 监控、评分、注入事件 | 教师后台 |

### 5.2 订单流转机制

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 采购专员  │───>│  仓库    │───>│   客服    │───>│ 调度员   │
│ (采购)   │    │ (验收入库)│    │ (接单)   │    │ (调度)   │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
                                                      │
       ┌──────────────────────────────────────────────┘
       ↓
┌──────────┐    ┌──────────┐    ┌──────────┐
│  司机    │───>│  仓库    │───>│ 客户签收  │
│ (运输)   │    │ (出库)   │    │ (完成)   │
└──────────┘    └──────────┘    └──────────┘
```

### 5.3 课堂使用方式

1. **分组：** 教师将班级分成若干小组（每组 5 人）
2. **角色分配：** 每组内部自行分配角色（或由教师指定）
3. **场景加载：** 教师从预设场景中选择一个，所有小组同时开始
4. **协作执行：** 各角色按流程完成自己的环节，通过系统实时传递订单
5. **观察与评分：** 教师实时监控，结束后查看系统评分和操作日志

## 六、界面设计规范

### 6.1 设计原则

- **简洁明了：** 避免过多装饰，突出操作区域
- **引导性强：** 新手模式下有操作提示和流程引导
- **状态直观：** 用颜色区分不同状态（蓝色=进行中、绿色=已完成、红色=异常）
- **信息层次：** 重要信息优先展示，次要信息可折叠

### 6.2 配色方案

| 用途 | 颜色 | 色值 |
|------|------|------|
| 主色 | Element Plus 蓝 | #409EFF |
| 成功 | 绿色 | #67C23A |
| 警告 | 橙色 | #E6A23C |
| 危险 | 红色 | #F56C6C |
| 信息 | 灰蓝 | #909399 |
| 背景 | 浅灰 | #F5F7FA |

### 6.3 页面布局

- **登录页：** 居中卡片式，选择角色和模式
- **主界面：** 左侧导航菜单（可折叠）+ 顶部工具栏 + 主内容区
