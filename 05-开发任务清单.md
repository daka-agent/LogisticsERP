# 物流教学软件 - 开发任务清单

> 本清单按阶段分解开发任务，可直接用于项目管理（如导入 TAPD、GitHub Projects 等）。
> 预计总工期：6周（单人开发），3-4周（2人协作）。

---

## 第一阶段：基础框架（第1-2周）

### 后端 - Python Flask

- [ ] **F-1.1** 初始化 Flask 项目（应用工厂模式、目录结构、配置文件）
- [ ] **F-1.2** 配置 SQLAlchemy + SQLite 数据库连接
- [ ] **F-1.3** 创建所有数据库模型（models/），包括 user, role, group, supplier, customer, vehicle, driver, goods, category, warehouse, zone, location
- [ ] **F-1.4** 编写数据库迁移脚本（或使用 Flask-Migrate）
- [ ] **F-1.5** 实现用户认证（Flask-Login + bcrypt 密码哈希）
- [ ] **F-1.6** 实现 RBAC 角色权限中间件（基于角色的路由装饰器）
- [ ] **F-1.7** 编写基础数据 CRUD API（客户、供应商、车辆、司机、商品、仓库/库区/货位）
  - [ ] 供应商 CRUD（`/api/suppliers`）
  - [ ] 客户 CRUD（`/api/customers`）
  - [ ] 车辆 CRUD（`/api/vehicles`）
  - [ ] 司机 CRUD（`/api/drivers`）
  - [ ] 商品 CRUD（`/api/goods`）
  - [ ] 仓库/库区/货位 CRUD（`/api/warehouses`, `/api/zones`, `/api/locations`）
- [ ] **F-1.8** 编写种子数据脚本（初始角色、示例供应商、示例商品等）
- [ ] **F-1.9** 统一 API 响应格式（`{ code, message, data }`）
- [ ] **F-1.10** 全局异常处理和错误码定义

### 前端 - Vue 3

- [ ] **V-1.1** 初始化 Vue 3 + Vite 项目，安装 Element Plus / Pinia / Vue Router / Axios
- [ ] **V-1.2** 配置 Axios 请求封装（baseURL、拦截器、Token）
- [ ] **V-1.3** 实现登录页面（用户名密码 + 身份选择 + 模式选择）
- [ ] **V-1.4** 实现主布局组件（左侧菜单 + 顶部导航 + 内容区）
- [ ] **V-1.5** 配置路由守卫（根据角色动态生成菜单）
- [ ] **V-1.6** 实现基础数据管理页面（表格 + 搜索 + 新增/编辑弹窗 + 删除确认）
  - [ ] 供应商管理页
  - [ ] 客户管理页
  - [ ] 车辆管理页
  - [ ] 司机管理页
  - [ ] 商品管理页
  - [ ] 仓库/库区/货位管理页（支持树形结构）
- [ ] **V-1.7** 前后端联调，确保基础数据 CRUD 全部可用

### 验收标准
- ✅ 系统可正常启动，浏览器打开可登录
- ✅ 基础数据（客户、供应商、车辆、商品、仓库）可正常增删改查
- ✅ 不同角色登录后看到不同菜单

## 第二阶段：采购 + 运输模块（第3-5周）

### 采购管理 - 后端

- [ ] **F-2.1** 采购申请 API
  - [ ] 创建采购申请（`POST /api/purchase-requests`）
  - [ ] 查询采购申请列表（`GET /api/purchase-requests`，支持分页和状态筛选）
  - [ ] 审批采购申请（`PUT /api/purchase-requests/:id/approve` / `:id/reject`）
- [ ] **F-2.2** 采购订单 API
  - [ ] 创建采购订单（选择供应商，关联审批通过的申请）（`POST /api/purchase-orders`）
  - [ ] 查询采购订单列表（`GET /api/purchase-orders`）
  - [ ] 更新订单状态（确认/发货/部分到货/完成）（`PUT /api/purchase-orders/:id/status`）
- [ ] **F-2.3** 采购到货验收 API
  - [ ] 创建验收记录（`POST /api/purchase-receipts`）
  - [ ] 验收通过后自动生成入库单（业务逻辑层处理）
- [ ] **F-2.4** 采购业务逻辑 Service（申请→审批→订单→验收→入库 的完整链路）

### 采购管理 - 前端

- [ ] **V-2.1** 采购申请页面（列表 + 新建申请弹窗 + 审批操作）
- [ ] **V-2.2** 采购订单页面（列表 + 新建订单弹窗 + 状态流转）
- [ ] **V-2.3** 到货验收页面（关联采购订单、录入验收信息）
- [ ] **V-2.4** 采购流程步骤条组件（显示当前环节）

### 运输管理 - 后端

- [ ] **F-2.5** 运输订单 API
  - [ ] 创建运输订单（`POST /api/orders`）
  - [ ] 查询订单列表（`GET /api/orders`，支持分页和多条件筛选）
  - [ ] 订单审核（`PUT /api/orders/:id/approve`）
  - [ ] 车辆调度（分配车辆和司机）（`PUT /api/orders/:id/dispatch`）
- [ ] **F-2.6** 运输跟踪 API
  - [ ] 添加跟踪记录（`POST /api/transport-records`）
  - [ ] 查询跟踪时间线（`GET /api/transport-records?order_id=xxx`）
  - [ ] 更新订单状态（发车/到达/签收/完成）（`PUT /api/orders/:id/status`）
- [ ] **F-2.7** 运费计算逻辑（按重量/体积/距离，支持手动调整）
- [ ] **F-2.8** 运输业务逻辑 Service（下单→审核→调度→运输→签收→结算 的完整链路）

### 运输管理 - 前端

- [ ] **V-2.5** 运输订单列表页（搜索 + 表格 + 分页）
- [ ] **V-2.6** 订单录入弹窗（客户信息、货物信息、起终点）
- [ ] **V-2.7** 车辆调度弹窗（选择车辆、选择司机、确认调度）
- [ ] **V-2.8** 运输跟踪页面（时间线展示运输状态变化）
- [ ] **V-2.9** 运输订单详情页（步骤条 + 基本信息卡片 + 操作记录）

### 验收标准
- ✅ 采购全流程可完整走通（申请→审批→订单→验收→生成入库单）
- ✅ 运输全流程可完整走通（下单→审核→调度→运输→签收）
- ✅ 运费自动计算正确
- ✅ 流程步骤条正确显示当前环节

## 第三阶段：仓储 + 库存模块（第6-8周）

### 仓储管理 - 后端

- [ ] **F-3.1** 入库管理 API
  - [ ] 创建入库单（`POST /api/inbound-orders`）
  - [ ] 入库验收（更新入库明细的数量和状态）（`PUT /api/inbound-orders/:id/inspect`）
  - [ ] 确认上架（选择货位，更新库存）（`PUT /api/inbound-orders/:id/shelve`）
- [ ] **F-3.2** 出库管理 API
  - [ ] 创建出库单（自动分配库存/货位，先进先出）（`POST /api/outbound-orders`）
  - [ ] 确认拣货（`PUT /api/outbound-orders/:id/pick`）
  - [ ] 确认发货（扣减库存）（`PUT /api/outbound-orders/:id/ship`）
- [ ] **F-3.3** 库内作业 API
  - [ ] 移库操作（`POST /api/inventory/transfer`）
  - [ ] 创建盘点任务（`POST /api/stocktake-tasks`）
  - [ ] 录入盘点结果（`PUT /api/stocktake-tasks/:id/count`）
  - [ ] 确认盘点差异（`PUT /api/stocktake-tasks/:id/confirm`）

### 仓储管理 - 前端

- [ ] **V-3.1** 入库管理页面（列表 + 验收操作 + 上架操作）
- [ ] **V-3.2** 出库管理页面（列表 + 拣货操作 + 发货确认）
- [ ] **V-3.3** 库内作业页面（移库、盘点功能）
- [ ] **V-3.4** 入库/出库详情页（步骤条 + 明细列表 + 操作记录）

### 库存管理 - 后端

- [ ] **F-3.4** 库存查询 API（`GET /api/inventory`，支持多条件组合查询）
- [ ] **F-3.5** 库存预警逻辑（入库/出库时检查是否触发预警）
- [ ] **F-3.6** 库存流水记录（所有库存变动自动写入 inventory_transactions）
- [ ] **F-3.7** 库存报表 API（周转率、呆滞库存、库龄分析）

### 库存管理 - 前端

- [ ] **V-3.5** 库存查询页面（多条件搜索 + 表格 + 库存状态标签）
- [ ] **V-3.6** 库存预警页面（预警列表 + 预警通知）
- [ ] **V-3.7** 库存流水页面（时间线/表格形式展示变动记录）
- [ ] **V-3.8** 盘点结果页面（差异报告展示）
- [ ] **V-3.9** 库存报表页面（用 ECharts 展示图表）

### 验收标准
- ✅ 采购→入库→库存增加 完整链路可走通
- ✅ 订单→出库→库存扣减 完整链路可走通
- ✅ 库存预警正确触发
- ✅ 盘点功能正常，差异报告准确
- ✅ 库存流水完整记录所有变动
