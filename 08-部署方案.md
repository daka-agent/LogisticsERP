## 二、部署方式对比

| 方式 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| **A. 单机部署** | 教师个人电脑演示 | 最简单，开箱即用 | 只能本机访问 |
| **B. 校园网部署** | 教学实验室/机房 | 学生统一访问，多人协作 | 需要服务器和网络配置 |
| **C. 云服务器部署** | 远程教学/在线课程 | 随时随地访问 | 需要云服务器费用 |
| **D. Docker一键部署** | 快速复制到多台机器 | 环境一致，迁移方便 | 需要Docker知识 |

**推荐组合：B（校园网） + D（Docker辅助）**

## 三、方案A：单机快速启动（开发/演示用）

### 前置条件
- Python 3.10+
- Node.js 18+
- Git

### 启动步骤

```bash
# 1. 克隆项目
git clone https://github.com/daka-agent/logistics-teaching.git
cd logistics-teaching

# 2. 后端启动
cd backend
pip install -r requirements.txt
python run.py          # 启动在 http://localhost:5000

# 3. 前端启动（新终端）
cd frontend
npm install
npm run dev            # 启动在 http://localhost:5173
```

### 访问
- 前端：http://localhost:5173
- 后端API：http://localhost:5000
- 管理员账号：admin / admin123

## 四、方案B：校园网/实验室部署（推荐）

### 硬件要求
- 服务器：4核CPU / 8GB内存 / 100GB硬盘
- 操作系统：Ubuntu 22.04 LTS 或 Windows Server 2022
- 网络：校园网内可访问

### 部署架构

```
┌──────────────────────────────────────────────┐
│              校园网 (192.168.x.x)             │
│  ┌────────┐  ┌────────┐       ┌────────┐     │
│  │ 学生电脑│  │ 学生电脑│  ...  │ 教师电脑│     │
│  └───┬────┘  └───┬────┘       └───┬────┘     │
│      └──────────┬┘                │           │
│         ┌──────▼──────────────────▼──┐        │
│         │     教学服务器              │        │
│         │  Nginx + Flask + Vue      │        │
│         │  192.168.1.100:80         │        │
│         └───────────────────────────┘        │
└──────────────────────────────────────────────┘
```

### Ubuntu 部署步骤

#### 1. 安装基础软件

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 Python
sudo apt install python3 python3-pip python3-venv -y

# 安装 Node.js
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install nodejs -y

# 安装 Nginx
sudo apt install nginx -y
```

#### 2. 部署后端

```bash
# 创建项目目录
sudo mkdir -p /opt/logistics-teaching
cd /opt/logistics-teaching

# 克隆代码（或上传代码包）
git clone https://github.com/daka-agent/logistics-teaching.git .

# 创建虚拟环境
cd backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 测试启动
python run.py
# Ctrl+C 停止
```

#### 3. 使用 Gunicorn + Nginx 部署（生产模式）

```bash
# 安装 Gunicorn
pip install gunicorn eventlet

# 创建 systemd 服务
sudo tee /etc/systemd/system/logistics-backend.service << 'EOF'
[Unit]
Description=Logistics Teaching Backend
After=network.target

[Service]
User=www-data
Group=www-data
WorkingDirectory=/opt/logistics-teaching/backend
Environment="PATH=/opt/logistics-teaching/backend/venv/bin"
ExecStart=/opt/logistics-teaching/backend/venv/bin/gunicorn \
    --worker-class eventlet \
    --workers 1 \
    --bind 127.0.0.1:5000 \
    "app:create_app()"

Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable logistics-backend
sudo systemctl start logistics-backend
```

#### 4. 部署前端

```bash
cd /opt/logistics-teaching/frontend
npm install
npm run build    # 生成 dist/ 目录
```

#### 5. 配置 Nginx

```bash
sudo tee /etc/nginx/sites-available/logistics-teaching << 'EOF'
server {
    listen 80;
    server_name 192.168.1.100;  # 替换为实际IP

    # 前端静态文件
    location / {
        root /opt/logistics-teaching/frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # WebSocket 支持
    location /socket.io/ {
        proxy_pass http://127.0.0.1:5000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}
EOF

sudo ln -s /etc/nginx/sites-available/logistics-teaching /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

#### 6. 防火墙配置

```bash
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
```

### 学生访问
- 浏览器打开：`http://192.168.1.100`
- 账号由教师通过后台创建和管理

## 五、方案C：云服务器部署（远程教学用）

### 推荐配置
- **腾讯云/阿里云** 轻量应用服务器
- 2核4GB / 50GB SSD / 3Mbps带宽
- 预估费用：约 50-100 元/月

### 额外步骤（在方案B基础上）

```bash
# 1. 域名解析（如有域名）
# A记录 → 服务器公网IP

# 2. 安装 SSL 证书（HTTPS）
sudo apt install certbot python3-certbot-nginx -y
sudo certbot --nginx -d your-domain.com

# 3. 自动续期（Certbot已自动配置）
sudo certbot renew --dry-run
```

## 六、方案D：Docker 一键部署

### 前置条件
- 安装 Docker + Docker Compose

### 启动步骤

```bash
# 克隆项目
git clone https://github.com/daka-agent/logistics-teaching.git
cd logistics-teaching

# 一键启动
docker-compose up -d --build

# 查看状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

### 访问
- http://localhost

### 常用命令

```bash
# 停止服务
docker-compose down

# 重启服务
docker-compose restart

# 查看数据库（进入容器）
docker-compose exec backend python -c "from app import db; ..."
```

## 七、数据备份策略

### SQLite 备份

```bash
# 手动备份
cp /opt/logistics-teaching/backend/instance/logistics.db \
   /opt/logistics-teaching/backup/logistics_$(date +%Y%m%d_%H%M%S).db

# 定时备份（crontab，每天凌晨2点）
0 2 * * * cp /opt/logistics-teaching/backend/instance/logistics.db \
   /opt/logistics-teaching/backup/logistics_$(date +\%Y\%m\%d).db

# 保留最近30天备份
0 3 * * * find /opt/logistics-teaching/backup -name "*.db" -mtime +30 -delete
```

### 数据恢复

```bash
# 停止服务
sudo systemctl stop logistics-backend

# 恢复数据库
cp /opt/logistics-teaching/backup/logistics_20260501.db \
   /opt/logistics-teaching/backend/instance/logistics.db

# 启动服务
sudo systemctl start logistics-backend
```

## 八、日常运维

### 查看服务状态

```bash
# 后端服务
sudo systemctl status logistics-backend

# Nginx
sudo systemctl status nginx

# 查看日志
sudo journalctl -u logistics-backend -f --no-pager
```

### 常见问题

| 问题 | 解决方案 |
|------|---------|
| 页面无法访问 | 检查 Nginx 和后端服务状态 |
| WebSocket 连接失败 | 检查 Nginx 的 socket.io 配置 |
| 数据库损坏 | 从备份恢复 |
| 端口被占用 | `lsof -i :5000` 查看占用进程 |

### 重置数据（教学场景）

```bash
cd /opt/logistics-teaching/backend
source venv/bin/activate
python reset_db.py    # 清空并重建数据库
```

## 九、安全建议

1. **修改默认密码**：首次部署后立即修改 admin 密码
2. **关闭调试模式**：确保 `FLASK_ENV=production`
3. **HTTPS**：生产环境务必启用 HTTPS
4. **定期更新**：保持系统和依赖包更新
5. **防火墙**：只开放必要端口（80/443）
